# VLT-5G 결과지 저장소 작업 지침

## 작업 시작 전 자동 확인 — 최우선

이 저장소에서 새 요청을 받으면 작업 형식 추정, 스킬 선택, 파일 생성·수정, Git 작업보다 먼저 적용 범위의 `AGENTS.md`를 끝까지 읽는다.

1. 현재 디렉터리부터 저장소 루트까지 상위 폴더를 확인한다.
2. 발견한 모든 `AGENTS.md`를 상위에서 하위 순서로 적용한다.
3. 현재 폴더에 파일이 없다는 이유로 지침이 없다고 판단하지 않는다.
4. 사용자가 `agent.md`, `agents.md`, `에이전트 지침`이라고 말해도 `AGENTS.md`를 뜻하는 것으로 해석한다.
5. 지침 확인 전에는 DOCX·PDF·HTML 등 산출물 형식을 임의로 정하지 않는다.

## 저장소와 브랜치

- 이 디렉터리가 실제 Git 저장소 루트다.
- 기본·배포 브랜치는 `master`다.
- GitHub Pages는 `master` 루트에서 배포된다.
- 작업 전 `git fetch origin`과 `git status -sb`로 원격 및 로컬 상태를 확인한다.
- 다른 작업자의 변경을 덮어쓰거나 `git reset --hard`, 강제 푸시를 사용하지 않는다.

## `학생ID 작성` 명령 해석

사용자가 `박시현12 작성`처럼 학생 식별자와 함께 `작성`이라고 요청하면 별도 형식을 묻지 않고 학생별 VLT-5G HTML 결과지 전체 작업을 실행한다.

기본 완료 범위는 다음과 같다.

1. 학생 VLT-5G 보고서, 녹취, 학생·학부모 검사지, 관리자 정보, 국어·수학·영어 APT를 확인한다.
2. 저장소 밖 로컬 완성본 HTML과 내부 운영 기록을 만든다.
3. 저장소 안에 학부모용·매니저용 각각의 중복되지 않는 ASCII 8자리 슬러그 폴더와 `index.html`을 만든다.
4. 관리자 목록에 두 링크를 추가한다.
5. 초기상담 Google Sheets의 올바른 학생 탭을 작성하고 재조회한다.
6. 해당 학생 관련 배포 파일만 커밋해 `origin/master`에 푸시한다.
7. 방금 푸시한 커밋의 Pages 빌드가 `built`인지 확인한다.
8. 학부모용·매니저용 URL이 HTTP 200이고 최신 학생 내용과 `noindex`를 포함하는지 확인한다.

사용자가 `초안`, `로컬만`, `배포하지 말 것` 등 범위를 명시하면 그 요청을 우선한다. DOCX·PDF는 사용자가 명시적으로 요구한 경우에만 만든다.

## 타입 확인

- 메인 타입과 히든 타입은 원본 보고서·녹취·로컬 타입 기준표를 대조한다.
- 번호만 보고 별칭을 추측하지 않는다.
- 학부모 HTML 상단에 `메인 TYPE N. 정식명칭`, `히든 TYPE N. 정식명칭`을 구분해 표시한다.
- 성장 방향 문구를 히든 타입 자리에 대신 넣지 않는다.
- 학부모용 HTML, 내부 운영 기록, 매니저용 HTML의 타입 표기가 일치해야 한다.

## 학부모용 HTML

- 표준은 `ifirlhkv/index.html`의 `REPORT_DATA` 기반 13개 섹션, 레이아웃, 카드·표·그래프, 모바일·인쇄 스타일이다.
- 학생별 데이터와 분석만 교체하고 임의 축약 템플릿을 만들지 않는다.
- VLT 세목의 `장점`, `보완점`, `발전할 점`을 빠짐없이 포집해 `검사 결과지 기반` 핵심 요약으로 연결한다.
- 국어·수학·영어 APT 상위 지표를 동일한 0~5 척도와 기준선으로 재구성한다.
- 점수 나열에서 끝내지 않고 `시각적 근거 → 현재 작동 방식 → 학습 리스크 → 실행 처방`으로 서술한다.
- 학부모용에는 관리자 링크, APT 원본 URL·문서 ID, 전화번호, 제작 과정, 자료 목록, 원본 확인 상태, 누락 변명을 넣지 않는다.
- 확인되지 않은 필수 자료를 추정해 채우지 않는다. 사용자에게 필요한 자료를 요청하거나 제외 승인을 받는다.

## Google Sheets — 박시은 탭 수준 유지

초기상담 기록지는 `박시은11_숭곡중_임` 탭을 셀 배치와 정보 밀도의 품질 기준으로 삼는다. 박시은의 학생 데이터는 복사하지 않고 대상 학생의 근거 자료만 사용한다.

1. 대상 학생 탭이 없으면 작업을 중단하지 않고 `샘플` 시트의 사본을 만든다. 탭 이름은 학생ID·실제 학교·담당자를 확인해 정확하게 지정한 뒤 작성한다.
2. 값을 쓰기 전에 대상 탭과 기준 탭의 같은 범위를 서식 포함 셀 읽기로 확인한다.
3. 병합 구조, 라벨 셀, 값 시작 앵커, 수식, 데이터 유효성 검사를 비교한다. 평문 범위 값만으로 입력 위치를 추측하지 않는다.
4. `학생이름 | 값`, `학부모 니즈 | 값`, `국어 | 분석`처럼 라벨 셀에 값을 합치지 않는다.
5. 라벨은 기존 라벨 셀에 보존하고 데이터는 지정된 흰색 값 셀의 좌상단 앵커에 쓴다.
6. 섹션 제목과 전략 헤더를 상세 내용으로 덮어쓰지 않는다.
7. 학습현황 진단은 과목별 사교육, 일정, 진도, 내신, 모의고사, 교재, 목표·특이사항을 근거 범위에서 구체화한다.
8. 확인되지 않은 값은 사실을 만들지 않고 `미확인`, `해당 없음`, `세부 요일·시간 미확인`처럼 상태를 적는다.
9. APT는 국어·수학·영어 라벨과 분석 셀을 분리하고 상위 지표, 강점, 우선 보완점을 함께 적는다.
10. 자기주도 진단은 기준 탭의 실제 병합 구조를 따른다. 별도 답변 앵커가 없는 질문 셀에 `|`로 답변을 붙이지 않는다.
11. 초기 3개월 전략은 헤더 행과 상세 내용 행을 분리한다.
12. 작성 후 같은 범위를 다시 읽어 라벨·값 분리, 섹션 제목, APT 3과목, 자기주도 진단, 전략 헤더·내용, 과목별 주요 칸을 검증한다.

## 공개·개인정보 원칙

- 저장소와 Pages는 공개 상태임을 전제로 한다.
- 학부모용 페이지에는 학생·학부모 연락처를 넣지 않는다.
- 매니저용 페이지와 관리자 목록은 내부 운영용으로 표시하고 `noindex, nofollow, noarchive`를 적용한다.
- `noindex`는 접근 제어가 아니므로 내부 링크를 학부모에게 공유하지 않는다.
- 학생 이름을 폴더명에 사용하지 않고 기존 학생 슬러그는 유지한다.

## 커밋·푸시·배포 검증

1. `git diff --check`를 실행한다.
2. 이번 작업 범위의 파일만 명시적으로 스테이징한다.
3. `git diff --cached --check`와 스테이징 내용을 검토한다.
4. `master`가 원격보다 뒤처졌으면 안전하게 원격 변경을 반영한다.
5. `git push origin master`를 실행한다.
6. 최신 Pages 빌드의 커밋이 푸시한 커밋과 같고 상태가 `built`인지 확인한다.
7. 변경된 모든 URL의 HTTP 200과 실제 내용을 확인한다.

완료 시 커밋 해시, Pages 상태, 검증한 URL을 보고한다.
