# VLT-5G 초기상담 결과지 배포

이 저장소는 에듀플렉스 길음점의 VLT-5G 초기상담 결과지를 GitHub Pages로 배포한다.

작업자는 어떤 수정이나 생성 작업보다 먼저 저장소 루트의 `AGENTS.md`를 끝까지 읽고 적용한다. 다른 로컬에서는 `master`를 pull한 뒤 이 파일을 동일한 기준 지침으로 사용한다.

저장소는 특정 로컬 절대경로에 의존하지 않는다. 새 PC에서의 clone, 비공개 입력자료 배치, 검증 및 배포 절차는 [`docs/PORTABLE_WORKFLOW.md`](docs/PORTABLE_WORKFLOW.md)를 따른다. 학생 원본과 인증정보는 공개 저장소에 올리지 않는다.

## 완료 기준

`결과지 작성`은 로컬 HTML 파일을 만드는 작업만 뜻하지 않는다. 아래 단계가 모두 끝나야 완료로 처리한다.

1. 학생·학부모 원본 검사지와 별도로 생성된 VLT-5G 보고서를 확인하고, 타입·검사일·13개 영역 원수치와 등급을 추출한다.
2. VLT-5G 보고서와 원장 상담 녹취 전체를 교차 분석한다. 수치와 타입은 VLT-5G 보고서, 실제 발언·합의·일정·금액은 상담 내용을 최우선 근거로 삼는다.
3. 학부모용 HTML을 작성한 뒤 VLT-5G 보고서 원수치와 모든 진단 항목이 일치하는지 다시 대조하고 공개 문구와 가격 정책을 검수한다.
4. Git 비추적 `private-output\[상담일]_[학생명]_내부관리기록.md`와 별도 8자리 영문·숫자 슬러그의 매니저용 HTML을 함께 작성한다.
5. 8자리 영문·숫자 랜덤 슬러그 폴더의 학부모용 `index.html`을 만든다.
6. 로고를 base64로 내장하고 학부모용·매니저용 HTML 모두에 `noindex, nofollow, noarchive` 메타 태그를 넣는다.
7. 관리자 배포 목록에 학부모용·매니저용 링크를 모두 연결하고 Git 비추적 `local-mapping\`의 두 배포 매핑을 함께 갱신한다.
8. 변경 범위만 커밋하여 `origin/master`에 푸시한다.
9. 해당 커밋의 GitHub Pages 빌드가 `built` 상태인지 확인한다.
10. 학부모용·매니저용 공개 URL이 모두 HTTP 200으로 응답하고 학생별 내용과 검색 차단 메타가 반영됐는지 확인한다.

내부 MD·매니저용 HTML·관리자 링크·푸시·두 공개 URL 검증 중 하나라도 끝나지 않았다면 결과지 작성은 완료가 아니다.

## 핵심 작성 원칙

- 상담 녹취가 학생별 사실관계의 최우선 근거다.
- 생성된 VLT-5G 보고서는 타입·검사일·13개 영역 수치와 등급의 최우선 근거다. 학생·학부모 원본 검사지와 VLT-5G 보고서를 같은 자료로 취급하지 않는다.
- 상담에서 확인되지 않은 합의·금액·일정은 확정해서 쓰지 않는다.
- 학부모용 페이지에는 제작 과정이 드러나는 표현을 노출하지 않는다.
- 학생 이름은 배포 폴더명에 넣지 않고 ASCII 랜덤 슬러그만 사용한다.
- 기존 학생의 결과지를 수정할 때는 기존 슬러그를 재사용해 전달된 링크를 유지한다.
- 관리자 목록의 내부 운영 링크는 학부모에게 공유하지 않는다.

## 배포 명령

```powershell
Set-Location <clone한 저장소 경로>
pwsh ./scripts/verify-repository.ps1
git diff --check
git status --short
git add -- <학생슬러그>/index.html <관리자목록>/index.html
git diff --cached --check
git commit -m "Deploy <student> counseling report"
git push origin master
```

배포 뒤에는 GitHub Pages 최신 빌드의 커밋이 방금 푸시한 커밋과 같은지 확인하고, 공개 URL을 직접 요청해 응답을 검증한다.

## 공개 전 필수 검사

- `file:///` 로컬 경로가 없는가
- 로고가 `data:image/...;base64,`로 포함됐는가
- `noindex, nofollow, noarchive`가 정확히 한 번 포함됐는가
- 공개 금지 제작 표현이 없는가
- 상담에서 확인한 학생 정보, 금액, 일정과 합의가 원문 근거와 일치하는가
- VLT-5G 보고서의 타입·검사일·13개 영역 원수치와 HTML이 일치하는가
- 이번 작업과 무관한 파일이 커밋에 포함되지 않았는가

상세 작성 규칙은 `AGENTS.md`, 공식 타입명은 `references/xcm16-types.md`, 학생별 비공개 입력·매핑 규칙은 `docs/PORTABLE_WORKFLOW.md`를 따른다.
